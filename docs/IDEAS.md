# Contest Ideas — What to Do

Brainstorm of **concrete things you can do** to strengthen your Live Agent / Creative Storyteller entry. Pick a few that fit your time and ambition.

---

## 1. Demo & presentation ideas (no new code)

**What to do:** Decide *how* you show the product so judges see ADK + Gemini Live + GCP.

| Idea | What to do | Why it helps |
|------|------------|--------------|
| **Scripted demo video** | Write a 90-second script: (1) “I’m uploading a video” → show upload; (2) “Our ADK pipeline on Cloud Run turns it into audio description” → show processing or completed AD; (3) “While I watch, I ask Gemini Live what’s on screen” → press G, ask 1–2 questions, get spoken answers. Record once, add captions. | Makes “Live Agent” story obvious in 90 seconds. |
| **Split-screen architecture** | In the same video or a slide: left = user flow (upload → watch → Live); right = “Under the hood: API Gateway → Cloud Run (FastAPI + Live proxy) | Cloud Run Job (ADK)”. | Highlights **Best Technical Execution & Agent Architecture**. |
| **Accessibility angle** | Demo with a narrator saying: “For someone who can’t see the screen, the AD track is the main experience; Live is for follow-up questions.” Optionally show screen reader or “audio-only” moment. | Strengthens **Best Innovation & Thought Leadership**. |
| **Before/after** | Short clip: 10 sec of raw video (no AD), then same 10 sec with AD on + one Live question. | Shows “creative storytelling” (Creative Storyteller) and multimodal UX. |

## 2. Quick product/demo tweaks (small effort)

**What to do:** Small changes that make the *submission* or *demo* more compelling.

| Idea | What to do | Where |
|------|------------|--------|
| **“Contest mode” or demo prompt** | Add a system instruction or “demo mode” that tells Gemini Live: “You are the ADAI video assistant. When the user asks about the video, use get_current_context and describe what’s on screen. Mention that you’re powered by Gemini Live and the video pipeline uses Google ADK on Cloud Run.” So when judges try it, the agent *says* the stack. | Backend: `gemini_live.py` system instructions or frontend prompt. |
| **One-click “Try Live” on homepage** | Ensure the homepage demo (HomePageVideoPlayer) has “Ask Gemini” very visible and works without login (or with a clear “Sign in to try Live”). | Frontend: CTA and visibility. |
| **Demo video list** | Curate 2–3 short, visually clear videos (e.g. person in a room, text on screen, action) so the demo always looks good. | Content / staging. |
| **Submission page** | Add a public “Built with Google ADK + Gemini Live” or “Powered by Google Cloud” line in footer or about page; link to your contest submission or demo. | Frontend: footer / about. |

## 3. Feature ideas that differentiate the Live Agent

**What to do:** Features that make the *agent* feel more “live” or more impressive.

| Idea | What to do | Effort | Impact |
|------|------------|--------|--------|
| **“Describe this moment”** | Button or voice: “Describe what’s on screen right now.” Call `get_current_context`, send to Gemini, speak the answer. You may already have this via tools; if so, add a visible button or shortcut. | Low | Clear “multimodal + Live” moment for the demo. |
| **“Jump to when…”** | User says “Go to when she picks up the phone.” Use existing `go_to_scene` / `search_video_context` and speak: “Found it, jumping there.” | Low (if tools exist) | Shows agent *controlling* the experience. |
| **Proactive line** | After 30 s of no speech, Gemini says one short line: “I’m here if you want to ask about the video.” (If Live API supports proactive audio.) | Medium | Feels more “agent-like.” |
| **Live + AD together** | When AD track is playing, let the user ask a question; Gemini answers in a gap or after the current phrase. | Medium | Strong **Creative Storyteller** + **Multimodal UX** story. |
| **“Compare with AD”** | Tool or prompt: “What would the audio description say for this scene?” and have Gemini generate one sentence in AD style (then optionally TTS). | Medium | Ties Live to your AD pipeline in the demo. |

## 4. Creative Storyteller–specific ideas

**What to do:** Emphasize “interleaved/mixed output” and narrative.

| Idea | What to do | Why |
|------|------------|-----|
| **Show the pipeline output** | In the demo or a second video: show a snippet of *generated script* (text) next to the same moment as *audio* (TTS). “This text was generated by Gemini; this is the same content as speech.” | Makes “interleaved text + audio” explicit. |
| **“Story so far”** | In the player or Live: “Story so far: [2–3 sentences].” Generated from scene titles or AD segments up to current time. | Positions the product as *narrative*. |
| **Refinement in the submission** | In the Creative Storyteller description, add: “Our ADK pipeline produces interleaved text (scene scripts, critic feedback, refinements) and audio (Cloud TTS); the final output is a single narrative stream (audio description) over video.” | Aligns with contest language. |

## 5. “Wow” ideas (more effort, aim higher)

**What to do:** Bigger bets for Grand Prize or Innovation & Thought Leadership.

| Idea | What to do | Why |
|------|------------|-----|
| **Live “director’s commentary”** | User turns on “Commentary mode”: Gemini Live acts as a director, describing shots, choices, and “what’s about to happen” in real time (using get_current_context + short lookahead if you have it). | Unique “creative storytelling” + strong demo. |
| **Multi-language Live** | User asks in Spanish; Gemini answers in Spanish (or same language as the AD track). You may already have language in Live; if so, demo it. | Innovation + accessibility. |
| **Pipeline status in the app** | After upload, show “ADK pipeline: Scene analysis → Critic → Refinement → Finalizing” with simple status steps. Link to “Our agents run on Google Cloud Run.” | Makes ADK and architecture visible to judges. |
| **Public “architecture” page** | One page: diagram + short bullets (API Gateway, FastAPI on Cloud Run, Gemini Live proxy, Cloud Run Job for ADK, Vertex AI). Link from submission. | **Best Technical Execution** proof. |

## 6. Checklist: pick 2–3 and ship

- [ ] **Demo video** (scripted, 2–3 min) — upload → ADK → playback + Live Q&A.  
- [ ] **Submission text** — use `submission_live_agent.md`; add 1–2 sentences from “Creative Storyteller” above if you enter that category.  
- [ ] **One “wow” moment** — e.g. “Describe this moment” button, or “Jump to when…” by voice, or pipeline status in UI.  
- [ ] **One visibility tweak** — “Built with ADK + Gemini Live” or demo prompt that names the stack.  
- [ ] **Architecture diagram** — one slide or page for Technical Execution.

Start with the **demo video** and **submission text**; then add one feature or visibility tweak so judges remember you.
